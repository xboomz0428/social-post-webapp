# Changelog

## v3.2.0 — 2026-06-22

### 儀表板與日曆修復

- **日曆顯示修復**：修正 `parseISO` 解析日期失敗時整個日曆變空白的問題，加入 try-catch 保護
- **儀表板「本月貼文數」修正**：改用 `scheduledAt` 優先判斷月份，排程中的貼文也能正確計入
- **儀表板日期解析強化**：所有圖表的日期計算加入 `isNaN` 檢查，避免無效日期導致資料消失
- **成效數據同步修復**：儲存貼文時的讚、留言、分享、觸及等數據現在會正確寫入 Google Sheets
- **新增提示**：觸及/互動率為零時顯示說明，引導使用者手動填入成效數據

## v3.1.0 — 2026-06-22

### 雲端備份管理面板

- **設定頁面新增「雲端備份管理」區塊**，四個操作按鈕：
  - 📥 **備份到電腦**：下載 JSON 檔，含帳號、口吻、貼文、AI 設定（完整備份）
  - 📤 **從電腦還原**：上傳 JSON 備份檔，合併到現有資料（不覆蓋）
  - ☁️ **全部寫入雲端**：一鍵推送所有帳號 + 口吻 + 貼文到 Google Sheets
  - 💻 **全部從雲端拉回**：一鍵從 Google Sheets 拉取所有資料到本機
- **備份檔案格式**：JSON，含版本號和匯出時間，可跨電腦搬移
- **AI API Keys 搬移提示**：說明 AI Keys 不存雲端，可用備份檔搬移

## v3.0.0 — 2026-06-22

### 雲端同步 — 跨電腦使用

- **所有資料自動同步到 Google Sheets**：
  - 帳號設定 → 「帳號」工作表（含 API Key JSON、排程規則 JSON）
  - 口吻設定 → 「口吻設定」工作表
  - 貼文 → 「貼文」工作表（原有 upsert 機制）
- **開啟 App 自動從雲端拉資料**（CloudSyncProvider）：
  - 每次開啟瀏覽器 session 自動拉一次
  - 雲端有但本地沒有的資料 → 自動匯入
  - 雲端和本地都有的貼文 → 以較新的 updatedAt 為準
  - 匯入後顯示 toast 提示並自動重新整理
- **帳號/口吻操作即時同步雲端**：
  - 新增帳號 → 同步
  - 編輯帳號 → 同步
  - 刪除帳號 → 雲端也刪除
  - 新增/編輯/刪除口吻 → 同步
- **API 擴充**（`/api/sheets`）：
  - 新增 `GET ?type=all` 一次拉回所有資料
  - 新增 `upsert-account`、`delete-account`、`upsert-style`、`delete-style` 操作
- **注意**：AI API Keys 因安全考量不存入雲端，換電腦需在「設定」頁面重新輸入

## v2.4.0 — 2026-06-22

### 台灣法規合規系統

- **AI 生成內建合規指令**：所有 AI 生成的文案在 system prompt 中注入台灣法規合規要求
  - 《藥事法》第 68、69 條 — 禁止非藥品宣稱療效
  - 《食品安全衛生管理法》第 28 條 — 食品不得宣稱療效
  - 《化粧品衛生安全管理法》第 10 條 — 化粧品不得宣稱醫療效能
  - 《公平交易法》第 21 條 — 禁止虛偽不實廣告
  - 內建安全替代用語對照表（治療→保養、療效→感受、消炎→舒緩...）
- **合規檢查 API**（`/api/ai/compliance`）：
  - 獨立的法規審查端點，使用低 temperature (0.1) 確保嚴謹
  - 回傳結構化結果：通過(safe)/警告(warning)/違規(violation)
  - 每個違規項目含：違規片段、對應法規、原因、修改建議
- **AI 生成後自動合規檢查**：生成完文案自動觸發合規審查，不用手動點
- **手動合規檢查按鈕**：編輯器新增「法規合規檢查」按鈕（橘色），可對任何貼文手動檢查
- **合規結果視覺化**：
  - ✅ 綠色框：合規通過
  - ⚠️ 黃色框：潛在風險，建議修改
  - ❌ 紅色框：明確違規，列出每項違規片段 + 法規 + 修改建議
